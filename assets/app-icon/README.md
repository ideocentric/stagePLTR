# App icon

Platform application icons for stagePLTR are generated by **iconsmaker** from a
single source SVG. Source and generated output both live here.

## Files

| Path | Committed? | Description |
|------|-----------|-------------|
| `master.svg` | yes | Source artwork — **1024×1024**, flat, text converted to outlines. **Add this.** |
| `symbolic.svg` | yes (optional) | 16×16 monochrome variant for Linux trays |
| `icons.toml` | yes | iconsmaker configuration |
| `generated/` | **yes** | Generated bundles (`.icns`, `.ico`, Linux `hicolor/` + `.desktop`/`.metainfo.xml`) |

> We **commit** `generated/` so normal builds and CI never need the Rust toolchain.
> Regenerate (below) whenever `master.svg` or `icons.toml` changes.

## Designing `master.svg`

- 1024×1024 square canvas.
- **Convert text to outlines** (no font dependencies — the renderer has no font loader).
- Design flat: **no** rounded corners, drop shadows, or gloss — iconsmaker applies the
  Apple squircle mask and depth bevel itself (tuned via `[effects]` in `icons.toml`).
- Keep it legible at 16×16 — test the smallest generated PNGs.

## Regenerating

Run from the **project root** (native Cargo — iconsmaker is a Rust tool):

```bash
cargo run --manifest-path /Users/matt/RustroverProjects/iconsmaker/Cargo.toml -- \
    --config assets/app-icon/icons.toml --verbose
```

Then inspect:
- macOS: open `generated/macos/stagePLTR.icns` in Preview
- Linux: `generated/linux/hicolor/256x256/apps/`
- Windows: `generated/windows/stagePLTR.ico`

Prerequisite: **Rust 1.85+** (edition 2024). See the Developer Guide for the full tool list.
